# 個人題目探索：C112181102

## 題目方向

海況觀測資訊APP

## 問題背景

海邊活動、搭船交通、港口作業與漁業活動都會受到海況影響，例如浪高、風速、風向、海溫與潮位等資訊。一般使用者雖然可以到氣象或政府開放資料平台查詢，但資料通常較分散，介面也不一定適合快速閱讀。若能透過手機 App 將常用海況資訊整合並視覺化，使用者就能更方便地掌握目前海邊或港口的安全狀況。

## 初步構想

本題目希望開發一款海況觀測資訊 App，整合即時海象觀測資料，讓使用者能依照測站或地區查看浪高、風速、風向、海溫、潮位與觀測時間等資訊。App 可用簡單清楚的介面呈現目前海況，並根據數據提供基本提醒，例如海況良好、需注意風浪或不建議進行海上活動。

## 可行功能

- 顯示各測站即時海況觀測資料
- 提供浪高、風速、風向、海溫、潮位等資訊
- 支援依測站或地區切換查詢
- 以顏色或圖示呈現海況風險等級
- 提供資料更新時間與手動重新整理功能
- 可加入搜尋功能，快速找到指定港口或測站

## 待討論事項

- 海況資料來源是否穩定，以及 API 是否有使用限制
- 風險等級判斷標準要如何設定
- 是否需要加入地圖顯示測站位置
- 是否需要支援推播通知或異常海況提醒
- App 主要使用對象要設定為一般旅客、海邊遊客，或港口相關工作者
